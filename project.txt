🏟️ Chapter 11 Team Programming Challenge
The Grand Tournament Manager
Scenario
A large regional gaming tournament is being hosted for multiple competitive teams. The tournament organizers have collected a large data file describing:
* Teams
* Players on each team
* Match results
* Player statistics

Unfortunately, the tournament software crashed and the data is stuck in raw text form. Your programming team has been hired to rebuild the analysis system that processes the tournament data and generates a full report.


Your task is to build a C++ Tournament Manager that reads the raw data file, organizes it using nested structures, analyzes the results, and generates an official tournament summary.
Your team must work together to design a modular program capable of processing large amounts of structured data.  You will be working with the file tournament_data.txt
________________


Program Requirements
Your program must demonstrate the following programming concepts:
   * Nested struct types
   * Use of pointers
   * Use of arrays or vectors
   * Extensive string manipulation
   * File input
   * File output
   * At ;east 14 functions (not including main())
   * Proper modular design
   * Collaboration among 7 team members



________________
Data Structure Overview
The program should organize the data using nested structures.
Example concept design:
Tournament
└── Teams
     └── Players
Possible structure relationship:
      * A Player belongs to a Team
      * A Team contains multiple Players
      * The Tournament contains multiple Teams

Each structure should contain appropriate fields such as names, statistics, and results.
Some structures must be accessed using pointers.
________________


Input File
Your program will read a file named:
tournament_data.txt
Example format:
BEGIN_TEAM
TEAM_NAME: Binary Beasts
WINS: 3
LOSSES: 1
PLAYERS: 3
PLAYER_DATA:
Alice|Sniper|18|6
Ben|Support|10|12
Charlie|Tank|9|8
END_TEAM

WINS: 4

Diana|Sniper|21|5
Ethan|Support|12|7
Frank|Tank|15|9
END_TEAM
Each player record contains:
PlayerName | Role | Kills | Deaths


Program Tasks
Your program must:
1. Read Tournament Data
Load the input file and parse each team and its players.
________________


2. Store Data Using Nested Structures
Example conceptual design:
struct Player
{
   string name;
   string role;
   int kills;
   int deaths;
};


struct Team
{
   string teamName;
   int wins;
   int losses;
   vector<Player> players;
};
The overall program should store multiple teams in a collection.
________________


3. Perform String Manipulation
Your program should perform several string operations such as:
         * Converting player names to consistent capitalization

         * Extracting values from "name|role|kills|deaths"

         * Searching for teams containing certain keywords

         * Cleaning extra whitespace

         * Formatting names for output

________________


________________
4. Calculate Statistics
Your program should calculate information such as:
For each team:
            * Total team kills
            * Total team deaths
            * Kill/Death ratio
            * Best player on the team

Across the entire tournament:
               * Team with the most wins
               * Player with the most kills
               * Average kills per player
               * Total players in the tournament
               * Highest kill/death ratio

________________


5. Generate a Tournament Report
Your program must generate an output file:
tournament_report.txt
The report should include:
                  * Team summaries
                  * Player statistics
                  * Overall tournament statistics
